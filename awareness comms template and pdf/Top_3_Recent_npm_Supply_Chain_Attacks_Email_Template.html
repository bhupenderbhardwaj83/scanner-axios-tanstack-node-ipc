Top 3 Recent npm Supply-Chain Attacks

Immediate Developer Workstation and Project Scan Advisory: TanStack, node-ipc, and Axios npm Supply-Chain Risks

Executive Summary

Recent npm supply-chain attacks have targeted widely used JavaScript/Node ecosystems. Affected components may exist inside projects, IDE extensions, package-manager caches, node_modules folders, and developer workstations. The objective is deterministic IOC/version-based detection, not generic keyword scanning. “No findings” means “no known indicators detected,” not a guarantee of no compromise.

Detection and Scanner Design

Create axios-tanstack-node-ipc.sh with two modes: Quick project scan for the current directory and Whole Mac scan for project, IDE, cache, application, and temp paths. It should produce section-wise console output, a timestamped readable log, and Critical/High/Medium/Info summary counts.

grep -E "@tanstack|node-ipc@|axios@(1\.14\.1|0\.30\.4)|plain-crypto-js" \
  **/package*.json **/package-lock.json **/pnpm-lock.yaml **/yarn.lock 2>/dev/null

Remediation

  • Stop using affected project temporarily.
  • Remove node_modules and clean npm/pnpm/yarn/bun caches.
  • Upgrade affected packages, regenerate lockfiles, and reinstall dependencies.
  • If a critical indicator is found, rotate GitHub tokens, npm tokens, SSH keys, AWS/GCP/Azure secrets, and CI/CD secrets.
  • Review shell history, build logs, and outbound connections.

Disclaimer: This scanner performs deterministic IOC and version-based verification against known malicious npm package releases and high-confidence supply-chain indicators. Absence of findings does not guarantee absence of compromise.
